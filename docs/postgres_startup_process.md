# شرح عملية تشغيل قاعدة بيانات PostgreSQL المضمنة

يقوم الكود في الدالة `_startPostgres()` داخل ملف `backend_manager.dart` بإدارة دورة حياة تشغيل قاعدة بيانات PostgreSQL المضمنة مع التطبيق. هذه العملية مصممة لتعمل بشكل مستقل وخفي (Background) دون تدخل من المستخدم. 

إليك تفصيلاً دقيقاً لكل خطوة تحدث داخل هذه الدالة:

## 1. التحقق من توفر الملفات التنفيذية
أول ما تقوم به الدالة هو التحقق مما إذا كان مجلد `postgres/bin` موجوداً أم لا (`if (postgresBinDir == null)`). 
إذا لم يكن المجلد موجوداً، فهذا يعني أن التطبيق ربما يعمل في وضع التطوير أو أنه تم بناء التطبيق بدون إرفاق ملفات قاعدة البيانات، وحينها يتخطى عملية التشغيل.

## 2. تجهيز مجلدات البيانات والسجلات (Logs)
```dart
Directory(pgDataDir).createSync(recursive: true);
Directory(pgLogDir).createSync(recursive: true);
```
لا يتم حفظ بيانات قاعدة البيانات بداخل مجلد التطبيق الرئيسي، بل يتم حفظها في المسار المخصص لبيانات المستخدم في نظام التشغيل (مثل `AppData` في الويندوز أو `.local/share` في لينكس). 
تقوم الدالة هنا بإنشاء مجلد `pgdata` (لحفظ الجداول والبيانات) ومجلد `pg_log` (لحفظ سجلات الأخطاء).

## 3. تهيئة قاعدة البيانات لأول مرة (initdb)
```dart
if (!Directory(pgDataDir).listSync().any((e) => e is File && e.path.endsWith('postgresql.conf'))) { ... }
```
يتحقق الكود مما إذا كان ملف الإعدادات `postgresql.conf` موجوداً. عدم وجود هذا الملف يعني أن هذه هي **أول مرة يفتح فيها المستخدم التطبيق** وأن قاعدة البيانات فارغة تماماً ولم يتم تهيئتها بعد.
في هذه الحالة، يتم استدعاء أداة `initdb`:
- تقوم هذه الأداة ببناء هيكل قاعدة البيانات الأساسي.
- يتم تعيين اسم المستخدم الافتراضي ليكون `postgres`.
- يتم إعطاء الصلاحيات المطلوبة وإنشاء ملف كلمة مرور مؤقت لتأمين الوصول الأولي.

## 4. تشغيل خادم قاعدة البيانات (pg_ctl start)
```dart
_postgresProcess = await Process.start(pgCtl, [ 'start', '-D', pgDataDir, ... ]);
```
بمجرد أن تكون البيانات مهيأة، يتم استخدام الأداة `pg_ctl` لتشغيل محرك قاعدة البيانات.
- يتم توجيه السجلات (Logs) لتُكتب في الملف `postgres.log` لتسهيل اكتشاف الأخطاء مستقبلاً.
- يتم تحديد المنفذ (Port) المخصص لقاعدة البيانات (افتراضياً 5433 لكي لا يتعارض مع أي قاعدة بيانات أخرى قد تكون مثبتة مسبقاً على حاسوب المستخدم).

## 5. مراقبة حالة التشغيل (Health Check)
لا يكفي فقط إرسال أمر التشغيل، فقاعدة البيانات تستغرق بضع ثوانٍ لتصبح جاهزة لاستقبال الاتصالات.
لذلك، يدخل الكود في حلقة تكرار (While loop) تستخدم أداة `pg_isready`.
تقوم هذه الأداة بعمل فحص مستمر (كل نصف ثانية) لمعرفة ما إذا كان خادم PostgreSQL قادراً على الرد.
- إذا رد الخادم، تخرج الدالة من الحلقة وتعتبر أن التشغيل قد نجح.
- إذا استغرق الأمر أكثر من الوقت المسموح به (`startupTimeout`)، تتوقف العملية ويتم الإبلاغ عن فشل التشغيل.

## 6. إنشاء قاعدة بيانات التطبيق (إن لزم الأمر)
بمجرد التأكد من أن خادم PostgreSQL يعمل بنجاح، تقوم الدالة أخيراً باستدعاء `_createDatabaseIfNeeded()`.
تقوم هذه الدالة الفرعية بتشغيل أداة `createdb` لإنشاء قاعدة البيانات الخاصة بتطبيقك (والتي تسمى `fingerprint_db`). 
إذا كانت قاعدة البيانات موجودة مسبقاً (من استخدام سابق للتطبيق)، فإن الأداة ستتجاهل الأمر وتستمر العملية بسلام.

---
**الخلاصة:**
هذه الآلية بالكامل تضمن أن المستخدم النهائي لا يحتاج إلى تثبيت PostgreSQL يدوياً، ولا يحتاج إلى تشغيل أي أوامر. التطبيق يتكفل بإنشاء الملفات، تهيئة النظام، تشغيل الخادم الخفي، وتجهيز قاعدة البيانات بمجرد النقر على أيقونة التطبيق.
